<objective>
Create the pure, TDD-driven full-file capture adapter `buildFullFileAnchor` — the
sibling of `buildDiffAnchor` for the full-file-at-commit view. It translates a
contiguous line selection over the flat full-file line list into a
`(source=FullFile, side=New, start_line, end_line)` anchor plus a plain-content
cached excerpt.

Purpose: This is the core net-new unit of Phase 68 and carries most of the risk
(D-01..D-04, L-01, L-04). Keeping it pure and fully tested lets the UI plan
(68-02) consume a trusted, deterministic capture result.

From src/lib/types.ts (review schema, frozen Phase 65):
```typescript
type Source = "Diff" | "FullFile";
type Side = "Old" | "New";

interface Anchor {
  commit_oid: string;
  file_path: string;
  source: Source;   // "FullFile" for this phase
  side: Side;       // always "New" for this phase
  start_line: number;
  end_line: number;
}
```

From src/lib/types.ts (diff line inputs the adapter reads):
```typescript
interface DiffLine {
  origin: "Add" | "Delete" | "Context";
  content: string;
  old_lineno: number | null;
  new_lineno: number | null;   // adapter reads ONLY this (D-02, L-01)
  spans: MergedSpan[];
}
interface DiffHunk { header: string; old_start: number; old_lines: number; new_start: number; new_lines: number; lines: DiffLine[]; }
interface FileDiff { path: string; status: DiffStatus; is_binary: boolean; hunks: DiffHunk[]; }
```

The full-file flat line list is produced by `file.hunks.flatMap(h => h.lines)`
(see src/components/diff/FullFileView.svelte:49) — the adapter operates on this
flat list, NOT on a single `file.hunks[hunkIdx]` like `buildDiffAnchor`.
