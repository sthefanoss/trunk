<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend → toast text (TrunkError.message) | Error and recovery messages from the backend are rendered in the UI; they must be displayed as text, not interpreted. |
| session-changed event payload → reload trigger | An event payload decides whether this panel reloads; a mismatched payload must be ignored, not acted on. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-04-XSS | Information disclosure / injection | toast rendering of backend messages | accept | TrunkError messages originate from this app's own Rust layer (IO errors, "not_open", "session_exists", D-15 recovery warning) — not from a remote/untrusted source. Svelte escapes interpolated text by default; no `{@html}` is used. Low risk for a local single-user tool. |
| T-65-04-EVT | Tampering | session-changed payload match | mitigate | The listener acts only when the payload matches this repo's canonical path (mirrors the `repo-changed` precedent); a stray/foreign payload is ignored, so another repo's mutation cannot force a reload here. |
| T-65-04-SC | Tampering | npm installs | mitigate | No new packages (RESEARCH Package Legitimacy Audit: N/A — uses existing `@tauri-apps/api`, testing-library). No install task; nothing to gate. |
</threat_model>
