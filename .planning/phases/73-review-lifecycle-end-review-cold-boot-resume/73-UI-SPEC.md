# Phase 73 — UI Design Contract

> Visual and interaction contract for the End-review affordance, cold-boot resume, session-summary line, and distinct empty-state copy added to `ReviewPanel.svelte`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase introduces **no new screens or components** — it adds three small UI affordances to the existing `ReviewPanel.svelte` header and list-render path. The contract below records the precise tokens, copy strings, and interaction states the planner and executor must hit.

| Role | Size | Weight | Line Height | Where used in this phase |

## Copywriting Contract

Word-level wording was explicitly deferred to UI-SPEC by CONTEXT.md D-06 ("structural decision is that the two states have distinct copy"). Below is the locked copy.

| Element | Copy | Source |
|---------|------|--------|
| End button — idle label | `End review` | CONTEXT D-03/D-05 (icon + verb-noun, matching `Copy` precedent) |
| End button — confirming label | `Click again to confirm` | CONTEXT D-05 (proposed string; locked here) |
| End button — title (idle, disabled when no session) | `No active review to end` | New — mirrors existing Copy button disabled-title pattern (`:393`) |
| End button — title (idle, enabled) | `(empty)` — no tooltip when enabled (matches Copy button behavior at `:393`) | Carry-forward |
| Session summary caption | `{N} comments · {M} commits` | CONTEXT D-04 — middle-dot separator U+00B7, no pluralization, no leading verb |
| Empty state — cold (`sessionState === 'none'`) heading | `No active review` | CONTEXT D-06 first line, polished from "No active review. Toggle review mode in the toolbar to start." |
| Empty state — cold body | `Toggle review mode in the toolbar to start.` | CONTEXT D-06 — split across two lines following existing `:418-422` two-span muted pattern |
| Empty state — warm (session active, zero commits) heading | `No commits in this review yet.` | EXISTING at `:419` — preserved verbatim. Reason: distinguishes a started-but-unseeded review from a started-and-seeded-but-zero-comments review (the existing branch already covers `groups.length === 0`). |
| Empty state — warm-with-commits-zero-comments heading | `Review started.` | CONTEXT D-06 polished from "Review started. Select diff lines or add a commit note to comment." |
| Empty state — warm-with-commits-zero-comments body | `Select diff lines or add a commit note to comment.` | CONTEXT D-06 |
| Toast — resume failed (newer_version or arbitrary IPC failure) | `Failed to resume review: {extracted message}` | RESEARCH Pattern 2 — uses `errorMessage(e, "Failed to resume review")` |
| Toast — end failed | `Failed to end review: {extracted message}` | RESEARCH Pattern 1 — uses `errorMessage(e, "Failed to end review")` |
| Toast — multi-tab end (Claude's discretion per CONTEXT) | **suppressed** — no toast; the cold empty state copy in tab B is the signal. | RESEARCH §Open Questions default recommendation |

**Empty-state branching summary (machine-readable for the executor):**

```
sessionState === 'none'                             → "No active review" / cold body
sessionState ∈ {'active','resume-available'}
  AND groups.length === 0                           → "No commits in this review yet." / existing body
  AND groups.length > 0 AND hasAnyComment === false → "Review started." / warm-with-commits body
  AND hasAnyComment === true                        → no empty state; list renders
```

The cold branch is **net-new**; the two warm branches preserve / refine existing strings. The `groups.length === 0` branch text stays verbatim because executor tests already assert it (`ReviewPanel.test.ts`).

**Destructive confirmation pattern:** inline two-step button (no modal). First click swaps label + color + background. Second click within ~3000ms invokes `end_review_session`. No-click within window → silent revert. Discipline: `clearTimeout` before each `setTimeout`; component-destroy cleanup clears the timer (RESEARCH Pitfall 3).

## Interaction States

Explicit state contract for the two new interactive elements (End button + empty-state regions). The cold-boot resume is a non-visible IPC sequence with no new visual states beyond the empty-state copy already enumerated.

### End button state machine

| State | Trigger | Text | Icon | Text color | Background | Border |
|-------|---------|------|------|------------|------------|--------|
| Hidden | `sessionState === 'none'` | (not rendered) | — | — | — | — |
| Idle | session present, `endConfirming === false` | `End review` | `Trash2` 14px | `--color-text-muted` | `transparent` | `1px solid --color-border` |
| Idle hover | mouse over / `:focus-visible` | `End review` | `Trash2` 14px | `--color-text` | `--color-hover` | `1px solid --color-border` |
| Confirming | first click, within 3000ms window | `Click again to confirm` | `Trash2` 14px | `--color-on-accent` (`#fff`) | `--color-danger-bg` | `1px solid --color-danger-border` |
| Confirming hover | mouse over confirming button | `Click again to confirm` | `Trash2` 14px | `--color-on-accent` | `--color-danger` (full red — heightened contrast for the destructive commit) | `1px solid --color-danger` |
| In-flight | second click → `await safeInvoke("end_review_session", ...)` | `Click again to confirm` (frozen during await) | `Trash2` 14px | unchanged from Confirming | unchanged | unchanged |
| Post-success | IPC resolves OK | (button hides — `session-changed` listener → `reload()` → `sessionState === 'none'` → state collapses to Hidden) | — | — | — | — |
| Post-failure | IPC rejects | reverts to Idle; `showToast(...)` fires; arrays untouched | reverts | reverts | reverts | reverts |

**Visibility decision (CONTEXT discretion + RESEARCH Pitfall 5):** hide entirely when `sessionState === 'none'`, do NOT render disabled. Reason: a disabled End button on a cold panel is dead weight — the affordance is conceptually absent. The Copy button's `disabled={!hasAnyComment}` pattern is different (a started session with zero comments is a valid state where Copy is meaningfully not-yet-usable).

**Icon:** `Trash2` from `@lucide/svelte` at 14px (matches the `Clipboard size={14}` precedent at `:399`). Verified present in `@lucide/svelte/dist/icons/index.d.ts` per RESEARCH §Environment Availability.

**No animation:** label/color swap is instant. Matches the existing Copy → ✓ Copied transition (no fade, no scale).

### Empty-state regions

| State | Container styling (existing pattern) | Heading | Body |
|-------|--------------------------------------|---------|------|
| Cold | `flex flex-col` `gap: 4px` `padding: 12px` (matches `:418`) | `font-size: 12px`, `color: var(--color-text)` | `font-size: 11px`, `color: var(--color-text-muted)` |
| Warm (no commits) | same | same | same |
| Warm (commits, no comments) | same | same | same |

No icon, no illustration in any empty state. Reason: the project's empty-state language elsewhere (existing `:418-430`) is pure text — adding a Lucide illustration here would set a new precedent for one specific panel.

### Session-summary caption

Single `<span>` rendered inside the scroll body, above the `{#if groups.length === 0}` block (RESEARCH Pattern 4: ABOVE the list, INSIDE the scroll body, NOT in the header). Visible only when `sessionState !== 'none'`; hidden in the cold branch.

| Property | Value |
|----------|-------|
| Tag | `<span>` |
| Container | direct child of scroll body; no wrapper |
| Padding | `2px 0` (carry-forward from `:537`) |
| Font size | 11px |
| Color | `var(--color-text-muted)` |
| Text | `{comments.length} comments · {commits.length} commits` |
| Visibility | gated on `sessionState !== 'none'` — hidden when cold; visible whenever a runtime/resumable session exists, including when both counts are zero |

When both counts are zero the line reads `0 comments · 0 commits` — accepted; the warm-empty heading sits below it and the combination remains coherent.
