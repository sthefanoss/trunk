**3. Compute row background style:**

The root `<div>` currently has:
```
style="color: var(--color-text); {selected ? 'background: var(--color-selected-row);' : ''}"
```

Replace the background logic with a function that handles search state:
```typescript
const rowBgStyle = $derived(() => {
  if (isCurrentMatch) return 'background: rgba(245, 158, 11, 0.2);';   // amber/orange strong highlight
  if (isSearchMatch) return 'background: rgba(234, 179, 8, 0.1);';     // yellow soft highlight
  if (selected && !isSearchActive) return 'background: var(--color-selected-row);';
  if (selected && isSearchActive) return 'background: var(--color-selected-row);';
  return '';
});
```

Actually, keep it simpler — use inline conditional in the style attribute:
```
style="color: var(--color-text); {isCurrentMatch ? 'background: rgba(245, 158, 11, 0.2);' : isSearchMatch ? 'background: rgba(234, 179, 8, 0.1);' : selected ? 'background: var(--color-selected-row);' : ''} {isSearchActive && !isSearchMatch && !isCurrentMatch ? 'opacity: 0.35;' : ''}"
```

This implements:
- **Current match:** amber/orange tint at 20% opacity (`rgba(245, 158, 11, 0.2)`)
- **Other matches:** yellow tint at 10% opacity (`rgba(234, 179, 8, 0.1)`)
- **Selected (non-search):** existing `var(--color-selected-row)`
- **Non-matching rows during search:** dimmed to 35% opacity
- **No search active:** normal appearance

**4. Update hover class:**
The existing `class:hover:bg-[var(--color-surface)]={!selected}` should also not apply during search highlighting:
```
class:hover:bg-[var(--color-surface)]={!selected && !isCurrentMatch && !isSearchMatch}
```

**Important:** Keep existing `onclick` and `oncontextmenu` handlers unchanged. Search highlighting is purely visual — row interaction stays the same.

- Empty input = no dimming, no highlights (per locked decision)

- Active matches = dimming + highlighting

**2. Add SVG overlay dimming:**

Find the SVG overlay section. There are three `<g>` groups for rails, connections, and dots. Each group needs opacity dimming when search is active.

Add a derived value for whether SVG dimming should be active:
```typescript
const searchDimmingActive = $derived(searchOpen && searchQuery.length > 0 && searchResults.length > 0);
```

For the SVG groups, we need to dim elements that belong to non-matching rows. Since the SVG paths span multiple rows (rails are coalesced), we can't easily dim individual paths. Instead, apply a global opacity reduction to the entire SVG and then un-dim matching row dots via a separate group.

**Simpler approach:** Apply a CSS opacity to the entire SVG overlay when search is active, then draw extra highlight dots/indicators for matching rows on top at full opacity. However, this adds complexity.

**Simplest approach (recommended):** Apply `opacity: 0.2` to the entire SVG overlay element when `searchDimmingActive` is true. The CommitRow dimming already handles per-row visual focus — the SVG dimming is supplementary. This is a single style change on the outer `<svg>` element.

On the SVG element (line ~749):
```svelte
<svg
  class="absolute top-0"
  width={refOffset + Math.max(graphColWidth, naturalGraphWidth)}
  height={contentHeight}
  style="left: 0; pointer-events: none; z-index: 1; {searchDimmingActive ? 'opacity: 0.2;' : ''}"
>
```

This dims all graph SVG elements (dots, rails, edges) uniformly when search is active, which provides the "full visual focus on matches" effect per the locked decision. The highlighted CommitRow backgrounds will stand out prominently against the dimmed graph.
