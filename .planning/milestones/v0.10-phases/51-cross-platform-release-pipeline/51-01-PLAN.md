Create `.github/workflows/release.yml` with the following exact structure:

**Trigger:** `on: push: tags: ['v*']` (per D-01, REL-01)

**Concurrency:** group `release-${{ github.ref }}` with `cancel-in-progress: true` (discretion, mirrors ci.yml pattern)

**Job name:** `build`

**Strategy:** `fail-fast: false` with matrix `include` containing 4 entries (per D-02):

| platform | args | rust_target |
|----------|------|-------------|
| `macos-latest` | `--target aarch64-apple-darwin` | `aarch64-apple-darwin` |
| `macos-15-intel` | `--target x86_64-apple-darwin` | `x86_64-apple-darwin` |
| `ubuntu-22.04` | (empty string) | (empty string) |
| `windows-latest` | (empty string) | (empty string) |

NOTE: D-13 specifies macos-13 for Intel, but research confirmed macos-13 was removed December 2025. Using `macos-15-intel` per research correction (last Intel runner, available until August 2027).

**Steps (in order):**

1. `actions/checkout@v6`

2. Linux system dependencies (conditional: `if: matrix.platform == 'ubuntu-22.04'`):
```bash
sudo apt-get update
sudo apt-get install -y \
  libwebkit2gtk-4.1-dev \
  build-essential \
  curl \
  wget \
  file \
  libxdo-dev \
  libssl-dev \
  libayatana-appindicator3-dev \
  librsvg2-dev \
  patchelf
```
Note: Same as ci.yml system deps PLUS `patchelf` (required for AppImage bundling).

3. `oven-sh/setup-bun@v2`

4. `bun install --frozen-lockfile`

5. `dtolnay/rust-toolchain@stable` with:
   - `targets: ${{ matrix.rust_target }}` (installs cross-compilation target when needed; empty string is fine for native builds)

6. `Swatinem/rust-cache@v2` with:
   - `workspaces: "src-tauri -> target"`
   - `save-if: ${{ github.ref_type == 'tag' }}` (save cache on tag pushes since releases are infrequent)

7. `tauri-apps/tauri-action@v0` with:
   - `env.GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`
   - `args: ${{ matrix.args }}`
   - `tauriScript: bunx tauri` (explicit bun usage per research pitfall 7)
   - NO `tagName`, `releaseName`, or `releaseId` (build-only mode per D-11)

Do NOT add post-build or upload steps yet -- Task 2 handles those.

  <action>
Add post-build steps to `.github/workflows/release.yml` AFTER the tauri-action step:

**Step 8: Create portable archive (macOS)**
- Condition: `if: runner.os == 'macOS'`
- Shell: default (bash)
- Commands:
```bash
cd src-tauri/target/${{ matrix.rust_target }}/release/bundle/macos
tar -czf trunk-macos-${{ matrix.rust_target }}.tar.gz *.app
```
Note: macOS cross-compiled builds output to `target/{rust_target}/release/bundle/` (not `target/release/bundle/`).

**Step 9: Create portable archive (Linux)**
- Condition: `if: runner.os == 'Linux'`
- Shell: default (bash)
- Commands:
```bash
cd src-tauri/target/release/bundle/appimage
tar -czf trunk-linux-x86_64.tar.gz *.AppImage
```
Note: Linux builds natively so output is in `target/release/bundle/`.

**Step 10: Create portable archive (Windows)**
- Condition: `if: runner.os == 'Windows'`
- Shell: `bash` (MUST specify `shell: bash` because Windows defaults to PowerShell -- per research pitfall 6)
- Commands:
```bash
cd src-tauri/target/release
tar -czf trunk-windows-x86_64.tar.gz trunk.exe
```
Note: Windows tar.gz wraps the standalone .exe binary from `target/release/`, not the installer from `bundle/nsis/`.

**Step 11: Upload artifacts**
- `actions/upload-artifact@v4` with:
  - `name: trunk-${{ matrix.platform }}-${{ matrix.rust_target || 'x64' }}` (unique per matrix entry per upload-artifact v4 requirement -- research pitfall 5)
  - `path:` multiline glob pattern:
```
src-tauri/target/**/release/bundle/**/*.dmg
src-tauri/target/**/release/bundle/appimage/*.AppImage
src-tauri/target/**/release/bundle/**/*.msi
src-tauri/target/**/release/bundle/**/*.tar.gz
src-tauri/target/**/release/*.tar.gz
```
Note: `**` glob handles both native builds (`target/release/`) and cross-compiled builds (`target/{triple}/release/`). The last line catches the Windows tar.gz which is in `target/release/` not `target/release/bundle/`.

  - `if-no-files-found: error` (fail if no artifacts produced -- catches path issues early)
  </action>
