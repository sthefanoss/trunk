<objective>
Add the End-review affordance to the ReviewPanel header: a visually distinct button next to Copy that requires a two-click inline confirmation before invoking `end_review_session`. Confirm pattern mirrors the existing Copy `setTimeout`/`clearTimeout` discipline at `ReviewPanel.svelte:130-134`; danger styling uses existing `--color-danger*` CSS custom properties (no inline colors). On confirmed End, the backend hard-deletes the on-disk file and emits `session-changed`; the existing listener at `ReviewPanel.svelte:353-367` re-runs `reload()` which sees `sessionState === "none"` and the panel collapses to the cold state.

Purpose: closes REQ-73-END (Phase 65 SC #4's "user has explicit end-review affordance" promise) so a review session is no longer implicitly permanent.

Output:
- New local runes: `endConfirming: $state(false)` and plain `endTimer: ReturnType<typeof setTimeout> | null = null` (mirrors Copy pattern).
- Three new handlers: `startEndConfirm()`, `cancelEndConfirm()`, `onEndClick()` — async, with `errorMessage`-based toast in the catch arm.
- `$effect` teardown clearing `endTimer` on component unmount (Pitfall 3).
- End button rendered inside the existing header `flex items-center` row, conditional on `sessionState !== "none"`, styled with `.end-button` + `.end-button.confirming` modifier classes using `--color-danger`, `--color-danger-bg`, `--color-danger-border`, `--color-on-accent` tokens. Icon: `Trash2` from `@lucide/svelte` at 14px.
- New `describe("End review", ...)` block in `ReviewPanel.test.ts` with `vi.useFakeTimers()` scoped to that describe (NOT promoted to the file-global beforeEach — the file's global `flush()` uses `setTimeout(r, 0)` and deadlocks under fake timers).
</objective>
