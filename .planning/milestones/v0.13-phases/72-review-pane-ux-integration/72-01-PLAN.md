<objective>
Collapse the review-session rune to two state axes (`reviewActive`, `rightPaneMode`), delete the preview-pane plumbing (`panelMode`, `previewMarkdown`, `showList`, `showPreview`), and make `generate(repoPath)` a pure async function returning the markdown string. This is the foundation Plan 03 (ReviewPanel Copy refactor) depends on — the Copy handler will consume `await session.generate(repoPath)` as a `Promise<string>`.

Purpose: Eliminate the state-machine third axis that exists only to host the deleted preview pane. Per CONTEXT.md design lock, the rune stops caching markdown — each Copy click re-invokes the IPC.
Output: A simplified `review-session.svelte.ts` rune with matching test coverage, plus a `Promise<string>` signature on `generate` that downstream Plan 03 will call.

This plan is a TDD plan (RED → GREEN → REFACTOR) per project standard (per-feature, single commit pair).
</objective>
