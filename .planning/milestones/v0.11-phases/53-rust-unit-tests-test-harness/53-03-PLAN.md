**File: `src-tauri/tests/common/drivers/branches.rs`**
Implement all 7 branch driver methods. Note: 5 of 7 require `&mut self` because they take `cache_map: &mut HashMap<String, GraphResult>`:
```rust
use crate::common::context::TestContext;
use trunk_lib::commands::branches;
use trunk_lib::error::TrunkError;
use trunk_lib::git::types::RefsResponse;

impl TestContext {
    // read-only — takes &self
    pub fn list_refs(&self) -> Result<RefsResponse, TrunkError> {
        branches::list_refs_inner(self.path(), self.state_map())
    }
    // read-only — takes &self
    pub fn resolve_ref(&self, ref_name: &str) -> Result<String, TrunkError> {
        branches::resolve_ref_inner(self.path(), ref_name, self.state_map())
    }
    // needs cache_map — takes &mut self
    pub fn checkout_branch(&mut self, name: &str) -> Result<(), TrunkError> {
        branches::checkout_branch_inner(self.path(), name, self.state_map(), self.cache_map())
    }
    // needs cache_map — takes &mut self
    pub fn delete_branch(&mut self, name: &str) -> Result<(), TrunkError> {
        branches::delete_branch_inner(self.path(), name, self.state_map(), self.cache_map())
    }
    // needs cache_map — takes &mut self
    pub fn rename_branch(&mut self, old: &str, new: &str) -> Result<(), TrunkError> {
        branches::rename_branch_inner(self.path(), old, new, self.state_map(), self.cache_map())
    }
    // needs cache_map — takes &mut self
    pub fn fast_forward_to(&mut self, branch: &str, target_oid: &str) -> Result<(), TrunkError> {
        branches::fast_forward_to_inner(self.path(), branch, target_oid, self.state_map(), self.cache_map())
    }
    // needs cache_map — takes &mut self
    pub fn create_branch(&mut self, name: &str, start_oid: &str) -> Result<(), TrunkError> {
        branches::create_branch_inner(self.path(), name, start_oid, self.state_map(), self.cache_map())
    }
}
```

IMPORTANT: The `checkout_branch_inner` and similar functions take both `state_map` (immutable) and `cache_map` (mutable). Since both are fields on `TestContext`, calling `self.state_map()` (returns `&HashMap`) and `self.cache_map()` (returns `&mut HashMap`) simultaneously requires careful borrowing. The driver implementation needs to pass `&self.state_map` and `&mut self.cache_map` directly (not through accessor methods) to avoid double-borrow issues. Read the actual field names in context.rs to get the direct field access right.
