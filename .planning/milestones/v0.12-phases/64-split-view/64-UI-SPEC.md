## Color

This phase uses the existing dark theme color system. No new primary colors are introduced. One new CSS custom property is added for phantom row backgrounds.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg: #0d1117` | Split view container background, context line background |
| Secondary (30%) | `--color-surface: #161b22` | Hunk header bars, file headers |
| Accent (10%) | `--color-accent: #388bfd` | Active toolbar segment background (`--color-accent-bg`), active toggle icon color |
| Diff Add | `--color-diff-add-bg: rgba(74, 222, 128, 0.1)` | Right panel: added line backgrounds |
| Diff Delete | `--color-diff-delete-bg: rgba(248, 113, 113, 0.1)` | Left panel: deleted line backgrounds |
| Phantom Row | `--color-diff-phantom-bg: rgba(139, 148, 158, 0.04)` | **NEW** -- empty spacer rows on whichever side has fewer lines |
| Destructive | `--color-danger: #f87171` | Discard Hunk / Discard Lines buttons |

Accent reserved for: active toolbar segments (content mode, layout mode), active toggle buttons (whitespace, invisibles, word wrap).

### New CSS Custom Property

```css
/* In app.css -- phantom row background for split view (D-07) */
--color-diff-phantom-bg: rgba(139, 148, 158, 0.04);
```

This is intentionally subtler than `--color-muted-bg` (0.1 opacity). The phantom row should be barely visible -- just enough to signal "this is a spacer, not content."

## Layout Contract

### Toolbar Layout (D-04, D-05)

```
[Hunk|Full] | [Rows2 icon | Columns2 icon] | [WS] [P] [Wrap] | filename | [Stage/Unstage] [x]
```

- Two segmented controls side by side, separated by a `toolbar-divider` (1px wide, 16px tall, `--color-border`)
- Content mode control: text labels "Hunk" and "Full" (existing style)
- Layout mode control: icon-only buttons using Lucide `Rows2` (inline) and `Columns2` (split) at 14px size
- Icon-only segmented control uses same `.segment` styles as text labels but with 4px horizontal padding instead of 8px
- Both controls use the existing `.segmented-control` flexbox wrapper
- Active state: `background: var(--color-accent-bg); color: var(--color-accent)` (existing pattern)

### Split View Panel Layout (D-08, D-09, D-10)

```
+------------------+----+------------------+
| Left Panel       | || | Right Panel      |
| (old content)    |drag| (new content)    |
| [gutter] [line]  |    | [gutter] [line]  |
+------------------+----+------------------+
```

- Container: `display: flex; flex-direction: row; height: 100%; overflow: hidden`
- Left panel: `flex: {splitRatio}; overflow-y: auto; overflow-x: auto; min-width: 0`
- Divider: 4px wide, `cursor: col-resize`, uses existing `pane-divider` visual style from RepoView
- Right panel: `flex: {1 - splitRatio}; overflow-y: auto; overflow-x: auto; min-width: 0`
- Default split ratio: 0.5 (50/50)
- Clamp range: 0.2 to 0.8 (20% minimum width per panel)
- Left panel gutter: old line numbers only (single column)
- Right panel gutter: new line numbers only (single column)
- No origin symbols (+/-/space) in split view (D-10) -- color backgrounds suffice

### Pane Divider (D-08)

Reuse the existing `pane-divider` CSS from RepoView.svelte:

```css
.split-divider {

    var(--color-border) 2.5px,

.split-divider:hover {
  background: linear-gradient(
    to right,
    transparent 1px,
    var(--color-accent) 1px,
    var(--color-accent) 3px,
    transparent 3px
  );
}
```

### Diff Line Rendering (per panel)

Each line in split view follows the existing diff line pattern minus origin symbols:

```
[gutter: line number] [content: merged spans with syntax + word-diff]
```

- Gutter width: dynamic, based on `max(String(maxLineNo).length, 1) + 1` characters (existing `gutterWidth()` function)
- Line style: `font-family: monospace; font-size: 12px; line-height: 1.5; padding: 0 8px; display: flex; align-items: flex-start`
- White-space: `pre` (default) or `pre-wrap` (when word wrap active)
- Background: `lineBackground(origin)` -- same function as HunkView/FullFileView
- Color: `lineColor(origin)` -- same function
- Syntax classes, word-diff emphasis, invisible characters, trailing whitespace: identical rendering to HunkView

### Phantom Rows (D-06, D-07)

- Empty div with no content, matching line height of 18px (12px font * 1.5 line-height)
- Background: `var(--color-diff-phantom-bg)` -- `rgba(139, 148, 158, 0.04)`
- No gutter numbers, no content, no origin symbol
- Height determined by the paired line on the opposite panel (when word wrap is active and a line wraps, the phantom must match that height)

### Hunk Headers in Split View (D-12, D-13, D-14)

**Split + Hunk mode:**
- Hunk header spans full width of both panels (interrupts the two-column layout)
- Background: `var(--color-bg)` (matches existing hunk header in HunkView)
- Left side: hunk header text (`@@ -x,y +a,b @@`) in `--color-text-muted` at 11px monospace
- Right side: Stage/Discard/Unstage buttons (same styles as HunkView)
- Padding: `4px 8px` with `8px` gap between elements

**Split + Full mode:**
- No hunk headers at all (D-13)
- Both panels show continuous document with change backgrounds

## Interaction Contract

### Synchronized Scrolling (D-11, VIEW-03)

- Scrolling either panel (vertical or horizontal) scrolls the other panel to match
- Implementation: `scrollTop` and `scrollLeft` mirroring with boolean `syncing` guard to prevent infinite recursion
- No independent scroll option -- always locked
- Works because both panels have identical row counts (phantom rows ensure 1:1 correspondence)

### Resizable Divider (D-08)

- `mousedown` on divider starts resize tracking
- `mousemove` on window: compute ratio = `(clientX - containerLeft) / containerWidth`
- Clamp ratio to [0.2, 0.8]
- `mouseup` on window: stop tracking, remove listeners
- Cursor changes to `col-resize` on hover and during drag

### Line Selection for Staging (D-15, VIEW-05)

- Line clicks are only interactive on the **right (new) panel**
- Only Add lines are selectable (Context lines are not)
- Click: select single line (toggle)
- Shift+click: range selection from last clicked line
- Selected lines: `background: var(--color-diff-add-bg-selected)` (existing `rgba(74, 222, 128, 0.25)`)
- When lines selected: hunk header shows "Stage Lines (N)" / "Discard Lines (N)" instead of "Stage Hunk" / "Discard Hunk"
- Cursor: `pointer` on selectable lines, `default` on non-selectable
- User-select: `none` on selectable lines to prevent text selection during click interactions

### Staging Disabled State (D-16)

- All staging buttons: `opacity: 0.4; cursor: not-allowed` when `ignoreWhitespace === true` or `hunkOperationInFlight === true`
- Title attribute: "Staging is disabled while whitespace changes are ignored"
- Pattern identical to existing HunkView staging guard

### Toolbar Interaction

- Content mode toggle: clicking "Hunk" or "Full" updates `contentMode`; active segment gets accent background
- Layout mode toggle: clicking Rows2 or Columns2 icon updates `layoutMode`; active segment gets accent background
- Both toggles are always visible regardless of the other's state
- Toggling layout mode does NOT change content mode and vice versa

## State Persistence

| Key | Type | Default | Store Function |
|-----|------|---------|----------------|
| `diff_content_mode` | `"hunk" \| "full"` | `"hunk"` | `getDiffContentMode()` / `setDiffContentMode()` |
| `diff_layout_mode` | `"inline" \| "split"` | `"inline"` | `getDiffLayoutMode()` / `setDiffLayoutMode()` |

### Migration from Old Key

The old `diff_view_mode` key may hold `"hunk"`, `"full"`, or `"split"`. On first load:
- `"hunk"` -> contentMode: `"hunk"`, layoutMode: `"inline"`
- `"full"` -> contentMode: `"full"`, layoutMode: `"inline"`
- `"split"` -> contentMode: `"hunk"`, layoutMode: `"split"`

After migration, the old key becomes dead data. New keys take priority.

## Accessibility Notes

- Toolbar segmented controls: buttons with `class:active` for visual state; no ARIA role needed (native button semantics)
- Layout mode icons: `title` attribute on each icon button for tooltip on hover
- Resizable divider: `cursor: col-resize` visual affordance; no keyboard resize support (matches existing pane-divider pattern)
- Selectable diff lines: `role="button"` on right-panel Add lines (matches existing HunkView pattern)
- Shift+click: `onmousedown` prevents text selection with `e.preventDefault()` (existing pattern)
