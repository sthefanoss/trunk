<objective>
Add word-level diff computation to the Rust backend. The `similar` crate computes inline changes for paired Delete/Add lines within hunks, populating the existing empty `word_spans` field on `DiffLine`. Performance guards skip computation for long lines (>500 chars) and dissimilar pairs (>60% edit distance).

Purpose: Backend produces word-span data so the frontend (Plan 02) can highlight specific changed words within modified lines.
Output: `similar` dependency added, `compute_word_spans_for_pair` and `compute_word_spans_for_hunk` functions, post-processing pass in `walk_diff_into_file_diffs`, integration tests.

    Add these two functions before `walk_diff_into_file_diffs`:

    **Function: `compute_word_spans_for_pair`**
    Takes `old_content: &str` and `new_content: &str`. Returns `(Vec<WordSpan>, Vec<WordSpan>)`.
    - Normalize both strings: if they don't end with `\n`, append `\n` (per D-02, `iter_inline_changes` requires newline-terminated lines from `from_lines`)
    - Create `TextDiff::from_lines(&old, &new)`
    - Iterate `diff.ops()`, for each op call `diff.iter_inline_changes(op)`
    - For each `InlineChange`, track byte offset (start at 0), iterate `change.iter_strings_lossy()` yielding `(emphasized: bool, value: Cow<str>)`:
      - Create `WordSpan { start: offset, end: offset + value.len() as u32, emphasized }`
      - Advance offset by `value.len() as u32`
      - Skip zero-length values
    - Match `change.tag()`: `ChangeTag::Delete` -> assign spans to del_spans, `ChangeTag::Insert` -> assign spans to add_spans, `ChangeTag::Equal` -> ignore
    - Return `(del_spans, add_spans)`

    **Function: `compute_word_spans_for_hunk`**
    Takes `lines: &mut [DiffLine]`. No return value (mutates in place).
    - Walk lines with index `i`, finding consecutive Delete runs followed by Add runs (per D-03, D-04):
      1. Skip non-Delete lines (increment i)
      2. Collect consecutive Deletes: record `del_start`, advance while `matches!(lines[i].origin, DiffOrigin::Delete)`
      3. Collect consecutive Adds immediately after: record `add_start`, advance while `matches!(lines[i].origin, DiffOrigin::Add)`
      4. Pair positionally: `pairs = min(del_count, add_count)`
      5. For each pair (del_idx, add_idx):
         a. **Length threshold (WORD-02):** if `del_content.len() > 500 || add_content.len() > 500` -> continue (leave word_spans empty)
         b. **Edit distance threshold (WORD-02):** `TextDiff::from_chars(del_content, add_content).ratio() < 0.4` -> continue (ratio < 0.4 means >60% different)
         c. Call `compute_word_spans_for_pair(del_content, add_content)`
         d. Assign returned spans to `lines[del_idx].word_spans` and `lines[add_idx].word_spans`

    **Step 3: Add post-processing pass to `walk_diff_into_file_diffs`.**
    After the existing `diff.foreach(...)` block and before the final `Ok(file_diffs.into_inner())`, add:

    ```rust
    let mut file_diffs = file_diffs.into_inner();

    // Pass 2: word-level diff enrichment
    for fd in &mut file_diffs {
        for hunk in &mut fd.hunks {
            compute_word_spans_for_hunk(&mut hunk.lines);
        }
    }

    Ok(file_diffs)
    ```
