<objective>
Fix shift+click text selection during line range selection in DiffPanel.

Purpose: UAT Test 2 reported that shift+click for range selection also triggers browser text selection. The root cause is that `e.preventDefault()` in the `onclick` handler fires too late -- the browser initiates text selection during `mousedown`, which fires before `click`.

Output: DiffPanel.svelte patched with an `onmousedown` handler that prevents default when shift key is held on selectable diff lines.
</objective>

In DiffPanel.svelte, find the diff line div element (around line 544-558) that renders each hunk line with the `onclick` handler.

Add an `onmousedown` handler to this same div that prevents browser text selection when the user is shift+clicking for range selection:

```
onmousedown={(e) => { if (isSelectable && e.shiftKey) e.preventDefault(); }}
```

This must be on the same div that has the `onclick` handler. The `onmousedown` event fires BEFORE `click`, so calling `preventDefault()` here stops the browser from initiating text selection before the click handler processes the range selection.

Do NOT remove the existing `e.preventDefault()` from the `handleLineClick` function's shift+click branch -- keep it as a safety net. The `onmousedown` handler is the primary fix; the existing onclick preventDefault is a secondary guard.

The `user-select: none` CSS property is already set on selectable lines (line 555), but this alone does not prevent shift+click text selection in all browsers. The mousedown preventDefault is the reliable cross-browser fix.
