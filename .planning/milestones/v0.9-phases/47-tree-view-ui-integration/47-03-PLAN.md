  <done>All 6 TREE requirements confirmed working by user in the running application.</done>
  <what-built>
    Complete tree view integration across all file list contexts with toggle, expand/collapse, keyboard navigation, and persisted preference.
  </what-built>
  <how-to-verify>
    Run `bun run dev` and open a repository with files in multiple directories.

    **TREE-01 — Toggle in staging panel:**
    1. Make changes to files in multiple directories (at least 2 levels deep)
    2. Look for a toggle icon button at the right edge of the "N files changed on main" header
    3. Click it — should switch from flat list (full paths) to tree view (directories + filenames)
    4. Click again — should switch back to flat list

    **TREE-02 — Toggle in commit diffs:**
    1. Click any commit in the graph to open CommitDetail in the right pane
    2. Verify the file list respects the same toggle — tree or flat matches the current mode
    3. Toggle the button in StagingPanel header — CommitDetail file list should also change

    **TREE-03 — Toggle in merge/conflict context:**
    1. If possible, create a merge conflict scenario (or check that the conflicted files section shows tree view when toggled)
    2. The conflicted files section uses the same TreeFileList component

    **TREE-04 — Expand/collapse with chevrons:**
    1. In tree mode, verify directory nodes show ChevronRight (collapsed) or ChevronDown (expanded)
    2. Click a directory row — it should toggle between expanded and collapsed
    3. Click the chevron icon — same behavior
    4. Verify directory text uses slightly bolder weight than file text

    **TREE-05 — State preserved across refreshes:**
    1. Expand some directories in the unstaged section
    2. Stage a file (click the + action button on a file)
    3. Verify the expanded directories remain expanded (tree does not collapse)
    4. Verify the staged file appears in the staged section

    **TREE-06 — Keyboard navigation:**
    1. Click inside a file list section to focus it
    2. Press ArrowDown — focus highlight should move to next visible row
    3. Press ArrowUp — focus highlight should move to previous row
    4. On a collapsed directory, press ArrowRight — directory should expand
    5. On an expanded directory, press ArrowRight — focus should move to first child
    6. On a file, press ArrowLeft — focus should jump to parent directory
    7. On an expanded directory, press ArrowLeft — directory should collapse
    8. On a file, press Enter — should open the file diff

    **Persistence:**
    1. Set tree view mode, close the app, reopen — tree view should still be enabled
  </how-to-verify>
