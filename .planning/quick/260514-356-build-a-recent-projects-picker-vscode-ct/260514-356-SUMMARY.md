## Tauri Command Added

`validate_recent_path(path: String) -> Result<bool, String>`

- Runs `repository::validate_and_open(&PathBuf)` inside `tauri::async_runtime::spawn_blocking`
- Returns `Ok(true)` iff the path exists AND opens cleanly via git2; any error maps to `Ok(false)`
- Outer `Err(String)` reserved for spawn-join failures; inner closure never errors
- Reuses the existing `validate_and_open` gate used by `open_repo`

## Decisions Made Under "Claude's Discretion"

- **Filter strategy:** case-insensitive substring (union of name + path).
- **No-op when same repo:** if `activeTab.repoPath === path`, close picker, no `open_repo` or `addRecentRepo`.
- **Silent prune on `open_repo` failure inside `handlePickerPick`:** removeRecentRepo + close, no toast.
- **A11y suppression** on `<li>` rows: keyboard nav handled at the input level (canonical command-palette pattern).
- **Storage uncapped, display capped:** the cap moved from the storage layer to the only consumer that wants it (WelcomeScreen). The picker now lists full history. (User asked for this after initial ship.)

## Post-Initial-Ship Iterations

User-driven follow-ups after the initial commit (`045de1f`):

- **`4ddb64a` — Scroll highlighted row into view on arrow nav.** Added `listEl` ref + `scrollIntoView({ block: "nearest" })` so keyboard navigation past the visible window scrolls the list rather than running the highlight off-screen.
