<decisions>
## Implementation Decisions

### Keyboard Shortcut
- **Bind to Cmd/Ctrl+R**, overriding the webview's default reload.
- Must call `preventDefault()` on the keydown event so the webview does not reload.
- Bind globally (window-level `keydown` listener installed once at app mount), not scoped to a component, so it works on both the WelcomeScreen and inside an open repo tab.
- Suppress the binding when an `<input>` or `[contenteditable]` element is focused (lets the user paste with Cmd+R modifiers in fields if relevant â€” defensive; harmless if no such fields care).

### Open Target
- Picking a recent opens it **in the current tab** (replaces whatever the active tab shows).
- Use the existing tab-open flow: call `safeInvoke("open_repo", { path })`, then route the tab to that repo identically to how `App.svelte:openRepoInTab` does today. Reuse, don't duplicate.
- If the picked repo path is the same as what's already in the active tab, treat it as a no-op (close the picker, don't reload).

### Missing Paths â€” Auto-remove on startup
- "On startup" here means: **every time the picker opens**, before rendering rows.
- Validate each `RecentRepo.path` via a Tauri command (new `path_exists` invoke, or reuse an existing FS check if one is already wired up). If the path is missing or not a git repo, drop it from the in-memory list and persist the pruned list via `removeRecentRepo` (or a batched equivalent).
- Pruning runs in parallel for all entries; render after the prune resolves.
- No user-visible "removed N entries" message â€” quiet by design.

### Row Content â€” Name + tildified path
- Two-line row: folder name (bold), full path below with `$HOME` collapsed to `~` via the existing `displayPath()` helper (`src/lib/path.ts`).
- No timestamp. No `lastOpenedAt` field added to `RecentRepo`.
- Hover state should match other clickable rows in the app (use existing CSS custom properties for accent/bg).

### Claude's Discretion
- **Picker layout**: centered modal overlay with backdrop, ~480px wide, fixed near top of viewport (VSCode-style command palette positioning). Use existing theme CSS custom properties â€” never inline colours.
- **Filtering**: case-insensitive substring match against both `name` and `path`. Optional fuzzy matching is nice-to-have but not required; substring is fine for a first pass.
- **Keyboard navigation inside the picker**: `â†‘`/`â†“` move the highlighted row, `Enter` opens it, `Esc` closes the picker, `Backspace` on an empty query closes the picker. Mouse hover also moves the highlight.
- **Empty state**: if recents is empty (or all pruned), the picker shows "No recent repositories" with a button to invoke the standard `open` dialog (mirroring `WelcomeScreen.openRepository`).
- **No-match state**: if the query has no matches, show "No matches".
- **Reopening when already open**: if the picker is opened a second time while already visible, the second Cmd/Ctrl+R is a no-op (do not double-toggle, do not steal focus mid-search).
- **Mount point**: add the overlay component near the top of `App.svelte`'s template so it sits above tab content. State (open/closed, query) lives in `App.svelte` or a dedicated `.svelte.ts` rune module â€” planner picks.
- **Component name**: `RecentReposPicker.svelte` (or similar) under `src/components/`.

</decisions>
