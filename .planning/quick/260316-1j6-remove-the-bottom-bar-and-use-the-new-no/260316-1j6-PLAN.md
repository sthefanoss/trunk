# Plan: Remove the bottom bar and use the new notification system for state updates

## Context

The StatusBar (bottom bar) displays remote operation progress, errors, and provides cancel/pull-now actions. The toast notification system already exists and is used by Toolbar for success/error messages. The StatusBar is redundant â€” its state feedback should go through toasts instead.

**Key insight:** `remoteState` must remain because Toolbar/PullDropdown use `isRunning` to disable buttons during remote ops. We only remove the StatusBar UI and relocate its unique responsibilities (remote-progress listener, cancel capability) into the Toolbar.

## Tasks

### Task 1: Move remote-progress listener and enhance Toolbar toast feedback

**files:** `src/components/Toolbar.svelte`, `src/components/PullDropdown.svelte`
**action:**
1. In `Toolbar.svelte`, add a `listen('remote-progress', ...)` effect (like StatusBar has) that updates `remoteState.progressLine`
2. In `Toolbar.svelte`'s `runRemote`, show a persistent/longer-lived toast with the progress line while `remoteState.isRunning` (or just rely on the final success/error toast â€” simpler approach)
3. In `PullDropdown.svelte`'s `runRemote`, add `showToast` calls for success/error (currently missing â€” only Toolbar's `runRemote` has them)
4. Move the `errorMessage()` helper from StatusBar into Toolbar (or a shared util) for detailed error messages in toasts

**verify:**
- `npm run check` passes
- Remote operations (pull, push, fetch) show toast notifications for success and error states
- Pull/push buttons still disable during remote operations

**done:** Toolbar handles all remote state feedback via toasts; PullDropdown shows toasts on completion
