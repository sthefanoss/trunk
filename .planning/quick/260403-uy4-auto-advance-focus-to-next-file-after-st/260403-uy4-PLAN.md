  <action>
1. Add an `advanceToNextFile` function to RepoView that:
   - Takes `(currentPath: string, section: "unstaged" | "staged" | "conflicted")` as parameters
   - Calls `safeInvoke<WorkingTreeStatus>("get_status", { path: repoPath })` to get fresh file lists
   - Gets the file list for the given section (e.g., `status.unstaged` for "unstaged")
   - Finds the index of `currentPath` in that list
   - If the file is NOT in the list (it was moved to another section by the action), pick the file that was at the same index position, or the last file if index exceeds length, or the first file if the list is non-empty
   - If a next file exists, call `handleFileSelect(nextFile.path, section)`
   - If no files remain in the section, call `clearStagingDiff()`

2. Modify the `onhunkaction` callback in the DiffPanel instantiation (around line 686-689). After calling `refetchFileDiff`, check if the returned `stagingDiffFiles` is empty (no hunks left) OR if all hunks in the returned diff have zero lines. If empty, call `advanceToNextFile(selectedFile.path, selectedFile.kind)`. The key insight: after `refetchFileDiff` completes, `stagingDiffFiles` is updated. Check if it represents an empty diff:
   - `stagingDiffFiles.length === 0` means the file has no diff content left
   - OR `stagingDiffFiles.every(f => f.hunks.length === 0)` means all hunks are gone
   In either case, auto-advance.

3. For whole-file stage/unstage from DiffPanel (`handleStageFile`/`handleUnstageFile` in DiffPanel call `onhunkaction`): After the `onhunkaction` call completes, the file has been fully moved to the other section. The `refetchFileDiff` will return an empty diff. The check added in step 2 will trigger `advanceToNextFile` automatically. No additional changes needed in DiffPanel for this path.

4. Refactor `handleFileResolved` (lines 208-227) to use `advanceToNextFile` instead of inline logic. Replace its body with simply calling `advanceToNextFile(selectedFile.path, "conflicted")` (which handles the null/error case by clearing). Keep the `if (!repoPath)` guard.

5. Add a new prop `onfileadvance` to the StagingPanel instantiation (line 732). Pass it as:
   ```
   onfileadvance={(path, kind) => {
     if (selectedFile?.path === path && selectedFile?.kind === kind) {
       advanceToNextFile(path, kind);
     }
   }}
   ```
   This allows StagingPanel to notify RepoView when a file action completes, and RepoView advances only if that file was the currently selected one.
