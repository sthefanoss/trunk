<objective>
Move the "review uncommitted changes" affordance from the Review panel into the working-tree diff line-selection toolbar (beside Stage/Unstage/Discard), and remove the v1 Review-panel entry point.

Purpose: v1 (quick 260531-4kk) shipped a working-tree-review backend + a Review-panel button. The user could not discover how to comment on uncommitted changes — the panel button + the internal "active session" concept were a dead-end. v2 puts the Comment affordance exactly where the user is already looking (the unstaged diff toolbar) and makes working-tree comments anchor to a get-or-create snapshot commit so earlier comments never orphan when the file changes mid-review.

Output: A discoverable in-diff Comment affordance for unstaged changes, a get-or-create snapshot backend (one command, never stacks redundant snapshots, never removes prior ones), and the v1 panel button removed.

THE DESIGN IS LOCKED (decided with the user + senior advisor). Do NOT re-litigate the snapshot-commit approach, the dual entry-point removal, or the New-side-only scope. Implement exactly as specified.
</objective>
